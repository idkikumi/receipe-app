# れしぴくん AI提案のWeb公開化 計画（2026-07-20）

現状の課題: Web公開版のAI提案は claude.ai の公開アーティファクトへ誘導する方式で、利用者が毎回claude.aiにログインする必要があり使う気になれない。デモ版（静的・固定レシピ）は快適だが本物のAI提案ではない。

## 結論

**デモ版と同じ使い勝手のまま、本物のClaudeがレシピを提案する構成にできる。利用者のログインは一切不要。費用は1回あたり約1円以下。**

## 仕組み

```
利用者のブラウザ（食材を選ぶ）
   ↓ fetch
Cloudflare Worker（堀さんのAPIキーを安全に保管・無料枠で運用可）
   ↓ Anthropic API
Claude Haiku 4.5 がレシピ3つをJSONで返す
   ↓
ブラウザがデモ版と同じUIで表示
```

- APIキーはWorkerの環境変数に置くのでブラウザ側には一切出ない
- ホスティングは既存のCloudflare構成のまま。Workerの無料枠は1日10万リクエストで十分

## 費用の見積り

Anthropic APIは前払いクレジット制（最低5ドル≒約750円のチャージから）。

| モデル | 1回の提案コスト目安※ | 100回使った場合 |
|---|---|---|
| Haiku 4.5（推奨） | 約0.8円 | 約80円 |
| Sonnet 5 | 約2.4円 | 約240円 |

※入力1,000トークン＋出力800トークン（レシピ3件のJSON）で概算。Haiku 4.5は $1/100万トークン（入力）・$5/100万トークン（出力）。

レシピ提案は定型タスクなのでHaiku 4.5で品質は十分。コミュニティに公開しても「べらぼうに課金」にはならない規模。

## 暴走・悪用への保険（実装に含める）

1. **max_tokensの上限固定**（1回の出力を制限）
2. **レート制限**: 同一IPからは1分に数回まで（Workerで実装）
3. **1日の総回数上限**: 超えたら「今日はお休み中」と表示（Workers KVでカウント）
4. Anthropicコンソール側でも月額の利用上限を設定できる

これで「最悪でも月◯円まで」と天井を決められる。

## 実装ステップ（builder向け・実行はGO後）

1. Anthropicコンソールで専用APIキーを発行（堀さんの操作。手順は当日案内）
2. Cloudflare Workerを作成: POST /suggest を受け、食材リストからHaiku 4.5へstructured outputs（JSONスキーマ固定）で問い合わせて返す
3. レート制限・日次上限を実装
4. フロント（index.html / React版）の「AI提案」ボタンをWorker呼び出しに差し替え。claude.aiアーティファクト誘導の画面は撤去
5. デモ版タブは「すぐ試せるサンプル」として残す
6. 動作確認→堀さんの確認→公開（pushは指示待ち）
